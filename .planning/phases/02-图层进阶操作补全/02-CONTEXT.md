# Phase 2: 图层进阶操作补全 - Context

## Domain
全面实现所有针对图层和图层组的高级操控、编组、合并与转化接口，支持大模型执行更复杂的图像编辑意图。

## Decisions

### 1. 目标图层指定机制
**决定：默认操作当前层，支持可选层名检索**
* 如果 AI 传入了特定的图层名称，程序会先在文档中检索该图层并将其设为活动图层。
* 如果未传入名称，则直接对当前的 `activeLayer` 进行操作。这符合人类交互习惯。

### 2. 图层移动接口设计
**决定：提供统一的 `move_layer` 接口**
* 支持传入 `(x, y, dx, dy)` 作为可选参数。
* 内部根据参数组合来自动判断是绝对坐标移动还是相对位移。这能减少暴露给 AI 的 Tool 数量。

### 3. 容错与静默处理
**决定：幂等处理（静默成功）**
* 操作尽可能做到幂等。如果图层已经是智能对象，再次调用转智能对象时直接返回成功；如果图层已经是栅格化，再次栅格化也返回成功。
* 防止不必要的报错打断连续的 AI 操作链。

### 4. 智能对象转换实现
**决定：彻底走 JSX 注入路线**
* 延续 Phase 1 的设计，对于 COM DOM 不支持的操作（如转智能对象），底层统一构建并注入 JSX 字符串来触发 ActionManager 指令。
* 避免在 Python 侧组装繁琐且不稳定的 ActionDescriptor。
