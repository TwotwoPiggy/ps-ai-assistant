# Phase 2 Discussion Log

**Q1: 目标图层指定机制：大模型调用改变不透明度、移动等接口时，如何确定目标图层？**
- 选定：(推荐) 默认操作当前选中图层，图层名称作为可选参数。如果传入了名称则先寻找并选中它，否则直接操作当前层。

**Q2: 图层移动接口设计：对于图层位移，我们应该如何提供 API？**
- 选定：(推荐) 提供单一的 move_layer 接口，支持可选参数 (x, y, dx, dy)。通过参数组合在内部判断是绝对坐标移动还是相对位移。

**Q3: 容错与静默处理：当遇到不合理的操作（如对背景图层尝试向下合并，或者对已栅格化的图层进行栅格化）时？**
- 选定：(推荐) 尽可能幂等处理（静默成功）。例如图层已经是栅格化或智能对象，再次转换直接返回成功。减少报错打断。

**Q4: 智能对象转换实现：Photoshop 的基本 COM DOM 不支持直接转为智能对象，需要底层动作调用。这部分实现方案是？**
- 选定：(推荐) 彻底走 JSX 注入路线。遇到 COM DOM 不支持的高级功能（如转为智能对象），直接组装 JSX 字符串并通过 ExecuteJavaScript 运行。
