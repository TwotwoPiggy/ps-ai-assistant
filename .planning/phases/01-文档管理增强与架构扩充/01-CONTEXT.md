# Phase 1: 文档管理增强与架构扩充 - Context

## Domain
在 `ps_tools` 中新增底层 ExtendScript 执行支持，并实现所有基础文档级操作（新建、打开、保存、色彩模式转换、历史记录控制等）。

## Decisions

### 1. JSX 脚本注入方式
**决定：采用直接内存传参 (`ps.DoJavaScript(string)`)**
* **理由：** 这是我为您推荐的方案。自动化脚本追求极致速度，直接传字符串无需读写磁盘，避免了各种可能的目录权限报错。
* **调试方案：** 为了防止出错后无法排查，我会在 Python 层加上 `logger.debug()`，把注入的 JS 字符串直接打印在控制台里，方便您随时查看。

### 2. 色彩模式弹窗拦截与安全确认
**决定：后端彻底静默，AI 负责安全提示**
* **底层实现：** 在执行任何可能弹窗的操作（尤其是转 CMYK）时，强行设置 `ps_app.DisplayDialogs = 3` (psDisplayNoDialogs)，绝对不允许 Photoshop 弹出任何原生警告打断自动化流程。
* **前端交互：** 为了尊重您的意愿，大模型的 `change_color_mode` Tool 提示词中会增加一条铁律：“执行此工具前，必须先在聊天界面提示用户即将进行色彩转换，等待用户明确回复『允许 (allow)』后才可调用。”

### 3. 保存与导出的默认路径规则
**决定：同级目录优先，桌面兜底**
* 当大模型调用 `save_document` 但没有传入绝对路径时，优先读取 `doc.FullName`（如果有）。
* 如果当前是一个刚刚新建的、还未存盘的“无标题”文档，则自动获取系统用户的“桌面”路径作为缺省导出目录。
