# Phase 05 Discussion Log

## 讨论 1: 无损调整与直接像素调整策略
- **选项**：A: 默认无损调整 / B: 默认破坏调整 / C: 人机确认
- **最终决定**：采用人机交互确认（C）。对模糊指令进行追问，并在无损模式下主动联动现有选区作为蒙版的提议。

## 讨论 2: 颜色输入格式与多模态扩展
- **选项**：A: 后端解析文字词汇 / B: 大模型转换 + 视觉上传提取
- **最终决定**：后端保持纯净，仅接收 RGB/Hex/HSB/CMYK。充分发挥大模型的多模态特性，支持图片传色与大模型自行提取。

## 讨论 3: 色阶 (Levels) 的深度支持
- **选项**：A: 仅支持主通道 / B: 支持红绿蓝三通道细分
- **最终决定**：选项 B，兼顾专业与稳定。API 开放全通道，但大模型组装参数时默认仅调节主通道，遇偏色纠正时启用细分。

## 讨论 4: 内容识别填充的兜底机制
- **选项**：A: 严格拦截并引导建立选区 / B: 自动检测透明图层轮廓
- **最终决定**：选项 A。以安全和准确为最高原则，严格拦截无选区的 CAF 操作并辅以大模型友好引导。
